## What This Is

A-share (China stock market) T+1 short-term trading analysis tool with a FastAPI web backend and mobile-first SPA frontend. Supports single stock analysis, full market scanning, ETF wave trading, backtesting, and signal calibration.

## Running the App

```bash
# Install dependencies (Python 3.11)
pip install -r requirements.txt

# Run web server (default port 10000)
uvicorn app:app --host 0.0.0.0 --port 10000

# CLI usage
python3 t1_trader.py                  # Full market scan
python3 t1_trader.py 002185           # Analyze single stock
python3 t1_trader.py --backtest       # Backtest strategy
python3 t1_trader.py --calibrate      # Signal weight calibration
python3 t1_trader.py --go             # T+1 quick decision (top 3)
python3 t1_trader.py --etf            # ETF wave scan
python3 t1_trader.py --etf-go         # ETF quick decision
```

Deploy: Render.com via `render.yaml` (Docker, port 10000).

## Architecture

### Core Pipeline (t1_trader.py, ~5700 lines)

The analysis flow is: **Data Fetch → Indicator Calc → Signal Detection → Weight Application → Multi-dimension Scoring → Risk Control → Recommendation**

Key stages:
1. `fetch_kline()` / `fetch_realtime_sina()` — K-line and real-time quotes from Tencent/Sina Finance
2. `calc_all_indicators()` — MA, MACD, KDJ, RSI, BOLL, volume ratio
3. `detect_signals()` — Pattern recognition across all indicator groups
4. `apply_weights()` — Apply signal weights (from `signal_weights.json` if calibrated)
5. `evaluate_signals_v2()` — Multi-dimension scoring: tech signals + capital flow + sector heat + sentiment + volume-price + K-line patterns + fundamentals + billboard + northbound + margin + streak analysis + shareholder increases
6. `calc_position_and_risk()` — ATR-based stop-loss, tiered take-profit, dynamic position sizing

Signal system uses **mutual exclusion groups** (MACD, KDJ, RSI, MA, BOLL, volume-price, patterns, capital flow) — only the highest-priority signal fires within each group.

Two trading modes: **T+1** (buy today, sell tomorrow) and **T+5** (wave trading, 1-5 day hold). ETF analysis has its own parallel pipeline (`evaluate_etf_trend`, `calc_etf_risk`, `scan_etf`).

### Web Layer (app.py)

FastAPI wrapping `t1_trader.py`. Uses SSE (`StreamingResponse`) for long-running scans (`/api/scan`, `/api/go`, `/api/go5`). Simple in-memory cache with TTL.

API endpoints:
- `/api/market` — Market sentiment + global markets
- `/api/sector` — Sector capital flow
- `/api/stock/{code}` — Single stock full analysis
- `/api/scan` — Full market scan (SSE streaming)
- `/api/go` — T+1 quick decision (SSE)
- `/api/go5` — T+5 wave decision (SSE)
- `/api/etf` — ETF scan (SSE)
- `/api/trades` — Trade tracking history

### Frontend (static/index.html)

Single-file mobile-first SPA with 8 tabs. Consumes SSE streams directly.

### Trade Tracker (trade_tracker.py)

Records recommendations to `trades.json`, auto-fetches D+1/D+5 closing prices, calculates actual P&L, and generates win-rate reports.

## Data Sources

All data is scraped from free public APIs (no API keys needed):
- **Tencent Finance**: K-lines (primary, stable)
- **Sina Finance**: Real-time quotes
- **East Money (东方财富)**: Capital flow, fundamentals, billboard (龙虎榜), northbound flow, margin data, global markets, commodities

Proxy environment variables are explicitly cleared at startup (direct connection to Chinese financial sites).

## Important Conventions

- All stock codes are 6-digit strings (e.g., `"002185"`, `"600519"`), zero-padded
- DataFrame columns are in Chinese: `日期`, `开盘`, `收盘`, `最高`, `最低`, `成交量`, `涨跌幅`
- `signal_weights.json` is auto-generated by `--calibrate`; do not hand-edit
- `jsonable()` in app.py handles numpy/pandas → JSON conversion (NaN → None)
- No test suite exists; verify changes via CLI analysis and backtest
